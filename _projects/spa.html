The Problem

UC Irvine's legal agreements management software (SPA) does not meet the needs of today's users.

SPA is expensive and difficult to maintain. It was written by a single developer in Visual Basic 6, and the code is not well structured. This makes fixing bugs or adding features nearly impossible, resulting in a frustrating experience for users. A very small number of people use SPA, and maintaining a legacy system just for them is no longer feasible for the University.

UC Irvine is planning on replacing SPA with a new system, but nobody had a complete picture of how it is used now to develop requirements for a replacement system.

Our Process

My team's objective was to gather information about current users and create a requirements document for a replacement system.

While members of my team didn't have formal titles, my role was that of project manager and UX designer. I communicated with our project sponsors, prepared presentations, and created the wireframes and most of the prototype.

Because some agreements in the system are confidential, we were not able to use it directly, so we interviewed users and observed how they used the system. We conducted 7 of these interviews on Zoom over the course of the project. We took notes and recorded each interview. We created a basic requirements document towards the beginning of the project, then filled in details and iterated on it as we gathered additional information.

When we had collected a list of features a replacement system would need to have, I created basic wireframes with Balsamiq. We showed these wireframes to users, who helped us identify missing features and improvements to make.

Based on these wireframes, we began to create interactive mockups using Figma.

The Result

My team produced a document detailing the requirements we had been gathering throughout the project for a replacement system, including a detailed description of features and other functional and nonfunctional requirements. I wrote the feature descriptions and significantly rewrote the finished document for consistency, completeness, and accuracy.

We also used Figma to create a basic clickable mockup to verify that our requirements would meet user needs. Other members of the team contributed towards early revisions of the mockups, but my work comprises most of the final version.
